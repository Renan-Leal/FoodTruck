<h1>Projeto Gerenciamento de FoodTruck</h1>
<div align="justify">
  <h2>Objetivo</h2>
  <p>O projeto simula uma aplicação de gerenciamento de foodtruck, onde  após fazer Login no sistema, o administrador conseguirá controlar o fluxo de pedidos, usuários,      produtos e gerar diferentes relatórios com base nos dados cadastrados. Foi um projeto pensado para aplicar lógica de programação, programação orientada a objetos e      modelagem de Banco de Dados. Durante as etapas e desafios, pude ter uma visão muito maior de como podemos aplicar técnicas de desenvolvimento para resolver              problemas do dia a dia.</p>
</div>
<div align="justify">
  <h2>Especificações Técnicas</h2>
  <p>A aplicação foi modelada com base no padrão arquitetural MVC e a conexão com o banco de dados foi perpetuada via JDBC. Para melhor preservação das boas práticas,        utilizamos nomes intuitivos nas classes e variáveis para pessoas com todos os níveis de conhecimento conseguirem entender o projeto de ponta a ponta. Como o projeto      foi pensado para fins didáticos, não criamos telas e todas inserções de dados são feitas através do console.</p>
</div>
<div>
  <h2>Ferramentas Utilizadas</h2>
  <ul>
    <li><b>SQL</b> - Linguagem de Banco de dados utilizada para modelagem do Sistema de Gerenciamento MySql;</li>
    <li><b>JAVA</b> - Linguagem de programação utilizada para perpetuação do CRUD no Banco de Dados;</li>
    <li><b>JASPER</b> - Ferramenta de modelagem de relatórios XML utilizado para gerar arquivos PDF. Com essa ferramenta, foi possível diferentes tipos de relatório,                             como por exemplo, relatório de lista de pedidos;</li>
   </ul>
</div>
